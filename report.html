注意: 高TP比率は期待値がマイナスに

TP=2.0:1以上ではWin Rateが急落し、多くの銘柄でExpectancyがマイナスに転じる。「利を伸ばす」はトレンドが強い銘柄のみで有効。

考察: 広いSL（2.0-2.5×ATR）+ 控えめなTP（0.5-1.0:1）が全体平均で最も安定。これはノイズによる不要な損切りを回避する効果がある。しかし、プロフィットファクターが1.0付近であり、取引コスト（スプレッド + 手数料）を差し引くとブレイクイーブンに近い。つまり、SL/TPの比率単体では優位性を生み出せず、エントリータイミング（シグナル品質）が真の勝敗を決めることを意味する。SL/TPは「優位性の保護」が役割であり、「優位性の創出」はシグナル側の仕事である。

23パターン検出戦略（ML）

市場データの大部分（~99%）はランダムノイズであり、再現性のあるパターンは全体の1%程度に限られる。従来の統計的戦略（Sec.22）が常時取引するのに対し、本セクションではRandomForestで「パターンが存在する瞬間」のみを検出し、高信頼度シグナルだけで取引するアプローチを検証する。

手法概要
  • 特徴量: 35個（価格構造8 + ボラティリティ6 + トレンド7 + 出来高4 + 時間5 + 上位時間軸5）
  • ターゲット: MFE/MAE方式 — 前方12barで有利方向2ATR超 & edge > 1.5ATRの場合に「パターン有り」
  • モデル: RandomForest（500本, 深さ8, class_weight=balanced）
  • 検証: 5-fold Purged Time-Series CV（purge=24bar, embargo=6bar）
  • 取引条件: パターン信頼度が閾値超（上位1-10%）の場合のみエントリー
  • 決済: SL=1.5×ATR, TP=2:1, Max Hold=12bar

24.13 分析的洞察

主要発見
  1. RF/Ensembleが最強: Mean Test Sharpe 1.61で並ぶ。ただし、RFは上位銘柄で突出し、Ensembleは全体的に安定。銘柄によって最適手法が異なるため、adaptive selection（銘柄ごとにCV実績で手法を選択）が最も有効。
  2. Entropy手法はTest>0が最多（33/55）: 非ランダム性検出は広範囲で機能。一方でSharpe絶対値はRF/HistGBTに劣る → フィルター（取引しない判断）として最適。

  5. 手法間の多様性が確認: 異なるパラダイム間のシグナル重複（IoU）が低い → アンサンブルの付加価値が大きい。
  6. USDJPY:全6手法で正Sharpe: 最もロバストな銘柄。Ensembleで5.3は実用的に有望。

28.5 Phase 2 Conclusions

  1. FFD features add marginal value at H1 scale: Most input series (volume, ATR, RSI) are already near-stationary. Only ffd_close has moderate importance (#16, 2.1%)
  2. Session features mostly useless: session_id, hours_to_session, is_high_vol_hour ranked #51-#59. The model already captures time effects via existing features (dow_sin/cos, hour_sin/cos)
  3. day_of_month_norm is highly valuable: Ranked #3 (5.6%) - captures month-end rebalancing effects. This single feature is trivially implementable in MQL5
  4. Adding noisy features hurts XGBoost: Sharpe drops 3.27 → 2.61 (-20%). Tree models are robust to noise but not immune. Feature selection > feature addition
  5. MT5 backtest skipped: Since 60 features shows no improvement in Python, building a new EA with 10 additional features is not justified
Recommendation: Keep 50-feature Pipeline-XGBoost from Phase 1 as best model. Consider adding only day_of_month_norm (single feature) in a future iteration. FFD may be more valuable at higher frequencies (M5, M15) or for non-stationary series like equity indices.
